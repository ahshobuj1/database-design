# Database Schema Planning

## Multi-Tenant SaaS DJ Platform

This document outlines the relational database structure designed to support multi-tenancy, subscription management, and dynamic portfolio generation.

---

### 1. Core Principles

- **Tenant Isolation:** Most tables include a `tenant_id` or link back to a `Tenant` to ensure data separation.
- **Scalability:** Use of `JSONB` (PostgreSQL) for theme-specific configurations to avoid frequent schema changes.
- **Auditability:** `created_at` and `updated_at` timestamps on all tables.

---

### 2. Table Definitions

#### 2.1 User & Identity

| Table: `users` | Type            | Description               |
| :------------- | :-------------- | :------------------------ |
| `id`           | UUID (PK)       | Unique identifier         |
| `email`        | String (Unique) | Login identifier          |
| `password`     | String          | Hashed password           |
| `role`         | Enum            | `SUPER_ADMIN`, `DJ`       |
| `is_verified`  | Boolean         | Email verification status |
| `otp`          | String          | Verification code         |
| `otp_expiry`   | Timestamp       | OTP validity window       |

#### 2.2 Theme Management

| Table: `themes`     | Type     | Description                            |
| :------------------ | :------- | :------------------------------------- |
| `id`                | Int (PK) | Unique identifier                      |
| `name`              | String   | e.g., "Neon Nights", "Classic DJ"      |
| `slug`              | String   | e.g., `neon-nights` (used for routing) |
| `preview_image_url` | String   | Thumbnail for the theme selector       |
| `default_config`    | JSONB    | Default layout/colors for this theme   |

#### 2.3 Tenant & Branding

| Table: `tenants` | Type            | Description                                 |
| :--------------- | :-------------- | :------------------------------------------ |
| `id`             | UUID (PK)       | Unique identifier                           |
| `user_id`        | UUID (FK)       | Links to `users.id`                         |
| `subdomain`      | String (Unique) | e.g., `alex-vibes`                          |
| `stage_name`     | String          | Public DJ name                              |
| `theme_id`       | Int (FK)        | Links to selected theme                     |
| `logo_url`       | String          | Path to branding logo                       |
| `bio`            | Text            | DJ description                              |
| `social_links`   | JSONB           | Links to Instagram, FB, etc.                |
| `config`         | JSONB           | Theme-specific settings (Colors, Hero text) |

#### 2.4 Subscriptions (SaaS Side)

| Table: `subscription_plans` | Type     | Description              |
| :-------------------------- | :------- | :----------------------- |
| `id`                        | Int (PK) | Unique identifier        |
| `name`                      | String   | Plan name (Basic, Pro)   |
| `price`                     | Decimal  | Monthly/Yearly cost      |
| `stripe_price_id`           | String   | Stripe identifier        |
| `features`                  | JSONB    | List of enabled features |

| Table: `subscriptions` | Type      | Description                      |
| :--------------------- | :-------- | :------------------------------- |
| `id`                   | UUID (PK) | Unique identifier                |
| `user_id`              | UUID (FK) | Links to `users.id`              |
| `plan_id`              | Int (FK)  | Links to `subscription_plans.id` |
| `stripe_sub_id`        | String    | Stripe subscription reference    |
| `status`               | Enum      | `active`, `past_due`, `canceled` |
| `period_end`           | Timestamp | Subscription expiration date     |

#### 2.5 Content Management

| Table: `mix_tapes` | Type      | Description                      |
| :----------------- | :-------- | :------------------------------- |
| `id`               | UUID (PK) | Unique identifier                |
| `tenant_id`        | UUID (FK) | Links to `tenants.id`            |
| `title`            | String    | Mix-tape name                    |
| `audio_url`        | String    | External link (SoundCloud, etc.) |
| `cover_url`        | String    | Cover image path                 |
| `order`            | Int       | Display sequence                 |

| Table: `events` | Type      | Description           |
| :-------------- | :-------- | :-------------------- |
| `id`            | UUID (PK) | Unique identifier     |
| `tenant_id`     | UUID (FK) | Links to `tenants.id` |
| `title`         | String    | Event name            |
| `description`   | Text      | Details about the gig |
| `event_date`    | Date      | Date of event         |
| `venue_name`    | String    | Location name         |
| `venue_address` | String    | Physical address      |

#### 2.6 Bookings & Financials

| Table: `bookings` | Type      | Description                        |
| :---------------- | :-------- | :--------------------------------- |
| `id`              | UUID (PK) | Unique identifier                  |
| `tenant_id`       | UUID (FK) | Links to `tenants.id`              |
| `client_name`     | String    | Customer name                      |
| `client_email`    | String    | Customer contact                   |
| `event_type`      | String    | e.g., Wedding, Club                |
| `status`          | Enum      | `pending`, `accepted`, `completed` |
| `total_amount`    | Decimal   | Negotiated fee                     |

| Table: `invoices` | Type      | Description                    |
| :---------------- | :-------- | :----------------------------- |
| `id`              | UUID (PK) | Unique identifier              |
| `tenant_id`       | UUID (FK) | For auditing purposes          |
| `booking_id`      | UUID (FK) | Null for subscription invoices |
| `user_id`         | UUID (FK) | Null for booking invoices      |
| `amount`          | Decimal   | Transaction value              |
| `type`            | Enum      | `SUBSCRIPTION`, `BOOKING`      |
| `method`          | Enum      | `STRIPE`, `CASH`               |
| `status`          | Enum      | `paid`, `unpaid`               |

---

### 3. Relationships

1.  **User -> Tenant:** One-to-One (Each DJ has one portfolio).
2.  **Theme -> Tenant:** One-to-Many (One theme can be used by many DJs).
3.  **Tenant -> Mix-tapes/Events:** One-to-Many.
4.  **Tenant -> Bookings:** One-to-Many.
5.  **Booking -> Invoice:** One-to-One.
6.  **User -> Subscription:** One-to-Many (History tracking).
